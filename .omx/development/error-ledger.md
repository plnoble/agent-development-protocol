### 2026-06-12 - PowerShell script blocked by execution policy

- Symptom: Directly invoking `scripts/init-agent-protocol.ps1` failed with a PowerShell execution policy error.
- Wrong assumption: The local shell would allow direct `.ps1` execution during verification.
- Root cause: The machine policy blocks unsigned PowerShell scripts by default.
- Detection method: Verification run failed before script logic executed.
- Fix: Run verification with `powershell.exe -NoProfile -ExecutionPolicy Bypass -File`.
- Prevention rule: When documenting or verifying this script on Windows, use the explicit `-ExecutionPolicy Bypass` invocation unless the user has configured script execution.
- Skill candidate: yes

### 2026-06-12 - Global SKILL.md written with UTF-8 BOM

- Symptom: `quick_validate.py` failed to read the global Codex `SKILL.md` with a `UnicodeDecodeError`.
- Wrong assumption: `Set-Content -Encoding UTF8` would be safe for skill files on this Windows setup.
- Root cause: Windows PowerShell can write UTF-8 with BOM, while the validation script read using the local default encoding.
- Detection method: Skill validation failed after global skill creation.
- Fix: Rewrote the global `SKILL.md` as ASCII and changed the initializer to write UTF-8 without BOM through .NET.
- Prevention rule: Use BOM-free UTF-8 or ASCII for skill and protocol files that will be parsed by local tooling.
- Skill candidate: yes

### 2026-06-12 - PowerShell template removed Markdown backticks

- Symptom: Files generated by `scripts/init-agent-protocol.ps1` lost Markdown inline-code backticks, and `worklog.md` contained literal `$ProjectName`.
- Wrong assumption: Double-quoted PowerShell here-strings would preserve Markdown backticks while still interpolating desired variables.
- Root cause: PowerShell treats backticks as escape characters, and escaped `$ProjectName` prevented interpolation.
- Detection method: A temporary generated project was inspected after the script appeared to pass basic file-creation checks.
- Fix: Switched templates to single-quoted here-strings with explicit `__PROJECT_NAME__` and `__DATE__` placeholders.
- Prevention rule: For PowerShell-generated Markdown templates, prefer literal templates plus explicit placeholder replacement.
- Skill candidate: yes

### 2026-06-12 - User prompt files initially read with wrong encoding

- Symptom: Chinese prompt text appeared as mojibake such as `浣犳槸`.
- Wrong assumption: `Get-Content -Raw` would decode the desktop prompt files correctly by default.
- Root cause: The files were UTF-8, while the initial PowerShell read used the default code page behavior.
- Detection method: The output contained recognizable UTF-8-as-ANSI mojibake; rereading with `-Encoding UTF8` produced correct Chinese.
- Fix: Re-read the prompt files with `Get-Content -Raw -Encoding UTF8`.
- Prevention rule: Explicitly set encoding when reading user-provided prompt libraries or Markdown files on Windows.
- Skill candidate: yes
